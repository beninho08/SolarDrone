You are designing a premium, human-crafted website for a drone-based 
solar panel inspection company. The result must feel like it was 
designed by a world-class studio — not generated by AI. 
Reference points: Arc.net, Linear.app, Stripe.com, but warmer and 
more grounded. Think "precision meets nature."

SCROLL STORYTELLING SECTION
(This is the centerpiece of the page — a sticky, narrative-driven 
scroll sequence that tells the full inspection journey)

CONCEPT:
A full-height sticky container takes over the screen for ~400vh 
of scroll. As the user scrolls, the LEFT side shows a persistent 
large visual (changes per chapter), and the RIGHT side reveals 
text chapter by chapter. A thin vertical progress line on the 
far left tracks how far the user is in the story.

CHAPTER 1 — "A helyszín" (The Site)
- Left visual: Aerial map view of a solar farm, 
  illustrated or photographic, top-down
- Right text fades in:
  Headline (serif, 48px): "Megérkezünk."
  Body: Short paragraph — the drone team arrives, 
  weather checked, equipment prepared.
- Ambient detail: A small drone icon appears at 
  bottom of visual, stationary

CHAPTER 2 — "A repülés" (The Flight)
- Left visual: transitions (crossfade) to a low-angle 
  drone-eye shot over solar panels
- Drone icon from Ch.1 begins moving slowly across 
  the visual in a grid pattern (SVG path animation, 
  scroll-driven)
- Right text fades in:
  Headline: "Minden sort végigpásztázunk."
  Body: The drone flies a precise grid, thermal camera 
  captures temperature anomalies invisible to the eye.

CHAPTER 3 — "A hiba" (The Finding)
- Left visual: Same panel field, but now a subtle 
  thermal overlay appears — 2–3 panels glow amber/red
- The reveal is slow: first normal view, 
  then thermal layer fades in as user scrolls deeper
- Right text fades in:
  Headline: "Amit szabad szemmel nem láthat."
  Body: Hotspots, bypass diode failures, soiling — 
  each identified, catalogued, GPS-tagged.
- Small annotation dots appear on the glowing panels 
  one by one (scroll-triggered)

CHAPTER 4 — "A jegyzőkönyv" (The Report)
- Left visual: Clean mockup of the report document — 
  a cream-colored PDF preview with the company logo, 
  panel map, and finding table visible
- Right text fades in:
  Headline: "48 órán belül a kezében."
  Body: A structured report lands in the client's 
  inbox. Every finding documented with photo evidence, 
  GPS coordinates, severity rating.
- Subtle animation: report pages stack in one by one

CHAPTER 5 — "A hozzáférés" (The Portal)
- Left visual: Portal UI mockup — code entry screen, 
  minimal card, warm background
- Right text fades in:
  Headline: "Egy kód. Azonnali hozzáférés."
  Body: No account needed. Client receives a unique 
  code by email, opens the link, downloads or views 
  the full report instantly.
- CTA button appears below body text:
  "Kérjen felmérést" — flat, dark, full-width of text column

END OF STICKY SECTION
- Scroll releases, page continues normally below

SCROLL STORYTELLING — INTERACTION SPECS
(for Figma prototype + developer handoff)

- Sticky container: position sticky, top 0, height 100vh
- Scroll distance per chapter: ~80vh each = ~400vh total
- Progress indicator: 2px vertical line, left edge, 
  fills with #C17D3C as user scrolls through chapters
- Chapter transitions:
  → Left visual: crossfade (opacity transition, 500ms ease)
  → Right text: fade up (opacity 0→1, translateY 24px→0, 
    400ms ease-out, triggered at chapter threshold)
- Drone path animation: SVG polyline, stroke-dashoffset 
  driven by scroll progress (chapter 2 only)
- Thermal overlay reveal: CSS clip-path or opacity layer, 
  scroll-driven (chapter 3 only)
- Annotation dots: scale 0→1 with 150ms stagger 
  (chapter 3, scroll-triggered)
- Report pages stack: translateY 40px→0 stagger on 3 pages 
  (chapter 4)
- All animations: ease-out only, NO bounce, NO spring physics
- Mobile: storytelling section becomes vertical scroll 
  (no sticky), chapters stack full-width
